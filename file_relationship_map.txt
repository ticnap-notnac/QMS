================================================================================
QFlow - System File and Function Relationship Map
================================================================================

This document maps the relations and dependency connections between frontend hooks,
API services, Express routes, controllers, services, and the database schema in QFlow.

--------------------------------------------------------------------------------
1. Front-End Core to API Network Routing
--------------------------------------------------------------------------------

[UI Components / Modals]
    │ (invokes handlers)
    ▼
[React Custom Logic Hooks] (src/hooks/)
    │ (calls REST methods)
    ▼
[Client API Services] (src/services/)
    │ (formats endpoints & appends Supabase JWT bearer token)
    ▼
[Vite API Client Wrapper] (src/lib/api.js)
    │ (dispatches HTTP requests)
    ▼
[Express Server Endpoint Routing] (server/routes/)

--------------------------------------------------------------------------------

4. Server Helpers and Utilities Context
--------------------------------------------------------------------------------

* authMiddleware.js (server/middlewares/authMiddleware.js)
    ├── Validates JWT tokens using Supabase Auth.
    └── Attaches decoded user credentials to `req.user`.

* roleMiddleware.js (server/middlewares/roleMiddleware.js)
    ├── Restricts backend access based on custom user roles.
    └── Enriches requests by attaching the database user record to `req.dbUser`.

* errorMiddleware.js (server/middlewares/errorMiddleware.js)
    └── Standardizes server exception payloads and catches controller errors via `next(err)`.

* date.js (server/utils/date.js)
    └── Utility converting active DateTime inputs cleanly to YYYY-MM-DD format.

* cbr.js (server/utils/cbr.js)
    └── Mathematical engine executing Jaccard string keyword similarities for case retrieval.

* constants (src/constants/index.js)
    └── Centralized, shared static dictionary configurations (roles, statuses, severities).

* reportsHelpers.js (src/utils/reportsHelpers.js)
    └── Framework-agnostic frontend business utilities extracted from React hook layers.
